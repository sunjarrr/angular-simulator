15. 1) ApiService - providedIn: 'root' потому что относится ко всему приложению, глобальное выполнение http запросов.
2) AuthService - providers компонента потому что отвечает только за авторизацию и текущего пользователя.
3) CartService - providers компонента потому что отвечает только за хранение товаров в корзине.
4) ProductFilterService - InjectionToken потому что сам по себе этот сервис может быть использован по своему на разных страницах а InjectionToken как раз дает нам эту гибкость как и говорилось в лекции.
5) NotificationService - useFactory потому что тут может быть использована условие с возвращением уведомления а это мы можем прописать в useFactory через () => {return ...}
6) ThemeService - providedIn: 'root' потому что отвечает за тему всего приложения глобально.
7) DashboardStatisticsService - providers компонента потому что ограничен одной страницой и после уничтожения не используется, это подходит под понятие компонента то что все что внутри компонента уничтожается вместе с ним.  
8) UserTableStateService - providers компонента потому что также узконаправленный сервис.
9) ModalService - providedIn: 'root' потому что у любого компонента должен быть доступ к этому сервису по условию. 
10) LoggerService - providers роутинга потому что есть связь с маршрутами и перенаправление, отправлением. 
11) AppConfig - providedIn: 'root' это самый главный корневой конфигуратор нашего приложения он обязательно должен быть рутовым.
12) CurrencyFormatter - useValue храним готовые форматы чисел а это у нас useValue.
13) AnalyticsService - InjectionToken потому что InjectionToken дает нам вариативность а тут сказано должен существовать только если включена аналитика.
